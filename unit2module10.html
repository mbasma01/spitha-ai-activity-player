The "Manual Labor" Trap

We know how to store data. But Data Science is about processing data. Doing this manually is slow, error-prone, and boring. We need machines.

Problem 1: Fixed Lists

Processing Items One-by-One

Scenario 1A: Manual Indexing

The Math Problem

Ratings: [7.8, 8.3, 7.9, 8.9].
Goal: Multiply every rating by 10.

ratings = [7.8, 8.3, 7.9, 8.9]

# Click button to type code...
The Solution

The For Loop

One machine handles infinite items.

ratings = [7.8, 8.3, 7.9, 8.9]

for score in ratings:
  print(score * 10)
Processed 10,000 items!

Scenario 1B: Data Cleaning

The Name Cleaner

Problem: You scraped a list of names, but they are all lowercase: ['alice', 'bob', 'charlie'].
Solution: Use a loop to apply .capitalize() to every single string.

names = ['alice', 'bob', 'charlie']

for name in names:
  clean_name = name.capitalize()
  print(clean_name)
> Alice
> Bob
> Charlie
Problem 2: Uncertainty

The "Until" Problem (While Loops)

Sometimes we don't have a list of items. We just have a Goal or a Condition. We need to keep working while the condition is true.

Scenario 2A: Manual Check

Repeated If Statements

Scenario: Sell tickets until seats run out. You have to check availability before every single sale.

seats = 20

if seats > 0:
  sell_ticket()
if seats > 0:
  sell_ticket()
# ... copy paste 18 more times ...
The Solution

The While Loop

"Keep selling WHILE seats are greater than 0."

seats = 20

while seats > 0:
  print("Ticket Sold!")
  seats = seats - 1
Seats Available: 20
Scenario 2B: Reaching a Target

The Savings Goal

Problem: You want to save $100 for a concert ticket. You save $10 a week. You need to keep saving while your money is less than 100.

savings = 0
# Loop until goal is reached
while savings < 100:
  savings = savings + 10
  print("Current: " + str(savings))

print("Goal Reached!")

Part 3: Common Loop Patterns

Pattern: Input Validation

The Security Gate

Problem: You need to keep asking for a password until it is correct.

user_input = ""
while user_input != "secret":
  user_input = input("Password?")

Manual IF statements would give up after 2 or 3 tries. The While loop never gives up.

Pattern: Accumulator

The Receipt

Problem: Calculate the total price of a cart with unknown size.

total = 0
for price in cart:
  total = total + price

Manual addition (p1+p2+p3) fails if the user removes an item. The loop adapts instantly.

Pattern: Search

The Needle in Haystack

Problem: Find if "Nemo" is in the list and stop looking.

for fish in ocean:
  if fish == "Nemo":
    print("Found him!")
    break

Manual checking requires reading every name yourself. The loop scans millions instantly.

The Golden Rule of Loops

  • Use a For Loop: When you have a container (List, Dictionary, DataFrame) and want to visit every item. ("For every movie...")
  • Use a While Loop: When you have a condition and don't know when it will end. ("While the user is typing...", "While seats are available...")
